# Design

## Chosen Approach

Provide a static example with one file per artifact type.

## Alternatives Considered

Use a real code change. Rejected because a minimal documentation example is easier to inspect.

## Module and File Impact

Only files under `examples/basic-change`.

## Data/API/Security Impact

None.

## Test Strategy

Manual documentation review.

## Rollback or Mitigation Notes

Remove the example files if they become misleading.
